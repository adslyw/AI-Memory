## 经验教训
- **渐进式重构的重要性:** 大规模目录结构重构应在有完整备份的情况下进行，并确保 git 能够正确跟踪移动操作（使用 `git mv` 而非手动移动+删除）。本次重构虽然 git 状态混乱，但核心文档内容得以保留。
- **文档结构设计原则:** 将相关文件按实体（Agent）组织而非按类型（所有 MEMORY.md 在一起）可以提高可维护性，尤其在多 Agent 系统中。每个 Agent 的完整上下文应保持自包含。
- **简化优于过度文档化:** MEMORY.md 的简化表明，过多的详细流程文档可能不如核心原则和规则实用。应保持记忆文件精华，将详细流程移至专门的文档（如 AGENTS.md）。
- **状态同步解耦:** 将 Star Office 的状态同步配置从根目录移至 agents/pm/ 和 skills/star-office-ui/ 目录，实现了关注点分离，使各组件更独立、更清晰。
